# Security Policy

## Supported Versions

Security updates and fixes are applied to the latest version of this repository.  
Older commits or forks are not actively maintained.

## Reporting a Vulnerability

If you discover a security issue in this project, please report it **privately** so that it is not disclosed to the public before a fix is available.

Please contact:

- Email: [kagan.durmus@tum.de](mailto:kagan.durmus@tum.de)  
- Subject: `[Security] Vulnerability in Beating-ETF-Buy-and-Hold`

When possible, include:

- A clear description of the issue and its potential impact  
- Steps to reproduce (code snippets, commands, or configuration)  
- Any relevant logs, screenshots, or proof of concept  
- Your environment details (operating system, Python version, etc.)

Please **do not** open a GitHub Issue or Pull Request that includes details of the vulnerability.

## Responsible Disclosure

When reporting, please:

- Do not abuse the vulnerability (for example, do not access, modify, or delete data you do not own, and do not attempt denial‑of‑service attacks).
- Avoid automated scanning that generates excessive traffic or load.  
- Allow a reasonable amount of time for the issue to be investigated and resolved before any public disclosure.

## Our Commitments

- Acknowledge your report within **3–5 business days**, whenever possible.  
- Keep you updated on the progress of the investigation and remediation.  
- Credit you in release notes or a security acknowledgements section, if you wish.
